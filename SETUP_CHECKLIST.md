# 帰宅後にやること

**結論から: `setup.ps1` を右クリックして実行するだけで、ほぼ全部終わります。**
この文書は、そのときに何が起きるかと、詰まったときの対処です。

---

## 先に知っておいてほしいこと

### これは何をする仕組みか
あなたのPCが1時間おきに勝手にClaude Codeを起動して、ゲームを少しずつ
改善し続けます。改善内容はGitHubに記録され、スマホからも見られます。

### 正直に言っておく注意点

**1. Claude Codeがあなたのフォルダを自由に書き換えます**
無人で動かすため確認をスキップする設定(`--dangerously-skip-permissions`)を
使っています。動くのはプロジェクトフォルダ内だけですが、
「勝手にファイルが変わる」ことは受け入れる必要があります。
変更は全部gitに記録されるので `natsu undo` でいつでも戻せます。

**2. PCの電源を入れっぱなしにする必要があります**
シャットダウンすると動きません。スリープからは自動で復帰する設定を
入れてありますが、電源を切るのはNGです。

**3. GitHubのリポジトリは必ずPrivateにしてください**
Publicにすると、ゲームの中身も要望に書いた個人的なことも全世界に公開されます。
セットアップ中に案内が出るので、そこで間違えないように。

**4. 週の利用制限を使い切ります**
希望通りの設定にしてありますが、自分でチャットを使いたい日に
使えなくなる可能性があります。そうなったら `natsu pause` で止めてください。

## セットアップ

### 準備するもの
- 使っているPC(Windows)
- GitHubのアカウント(持っているとのことなのでOK)
- 30分くらいの時間

### 手順

**1. フォルダを作る**

`D:\natsu-project` を作って、以下を全部その中に入れる:

```
natsu_no_tonari.html   ← ゲーム本体。これを忘れがちなので注意
setup.ps1
natsu.ps1
natsu.bat
auto_improve.ps1
CLAUDE.md
REQUESTS.md
BACKLOG.md
PROGRESS.md
IMPROVEMENTS_LOG.md
COMMANDS.md
```

**2. setup.ps1 を右クリック →「PowerShell で実行」**

これで自動セットアップが始まります。あとは画面の指示に答えるだけ。

「このスクリプトは実行できません」のような警告が出た場合は、
PowerShellを開いて次を実行してください:

```
cd D:\natsu-project
powershell -ExecutionPolicy Bypass -File .\setup.ps1
```

**3. 途中で聞かれること**

| 聞かれること | 答え方 |
|---|---|
| Gitの名前・メール | ローマ字の名前と、GitHubのメールアドレス |
| Claude Codeを入れるか | `y` |
| ログイン確認 | `y` → ブラウザでログイン → 画面が出たら `/exit` |
| GitHubのURL | 案内通りにリポジトリを作ってURLを貼り付け(**Privateで!**) |
| natsuコマンドの設定 | `y` |
| 何時間おきか | 何も入れずEnter(1時間) |
| タスクを登録するか | `y` |

**4. 終わったらPowerShellを開き直して、確認**

```
natsu doctor
```

全部緑の [OK] になっていれば完成です。赤い [NG] があれば下の表を見てください。

```
natsu now
```

1回だけ手動で動かします。少し待って何か起きればOK。

```
natsu play
```

ゲームが開けば全部つながっています。

## セットアップが失敗したとき

**画面のスクショをClaudeのチャットに送ってください。** それが一番早いです。

よくあるやつ:

| 症状 | 原因と対処 |
|---|---|
| `node が見つかりません` | https://nodejs.org/ のLTS版を入れてPC再起動 |
| `git が見つかりません` | https://git-scm.com/download/win を入れてPC再起動(選択肢は全部Next) |
| `claude` を入れた直後に認識されない | PowerShellを閉じて開き直す |
| git push で止まる/失敗する | GitHubのログインウィンドウが裏に隠れていないか確認 |
| GitHubのパスワードが通らない | GitHubは今パスワードでは入れません。設定→Developer settings→Personal access tokens でトークンを作り、それをパスワード欄に貼る |
| タスク登録に失敗 | 下の「手動でタスクを登録する場合」へ |

## 手動でタスクを登録する場合

setup.ps1 の自動登録が失敗したときだけ。

1. Windowsキー →「タスクスケジューラ」
2. 右メニュー「タスクの作成」(基本タスクではなく**タスクの作成**)
3. **全般**タブ
   - 名前: `NatsuNoTonari_AutoImprove`
   - 「ユーザーがログオンしているときのみ実行する」を選ぶ ←重要
4. **トリガー**タブ →「新規」
   - 開始: 「1回」
   - 「繰り返し間隔」にチェック → `1時間` / 継続時間 `無期限`
5. **操作**タブ →「新規」
   - プログラム: `powershell.exe`
   - 引数: `-ExecutionPolicy Bypass -WindowStyle Hidden -File "D:\natsu-project\auto_improve.ps1"`
6. **条件**タブ
   - 「AC電源に接続時のみ」のチェックを**外す**
   - 「タスクを実行するためにスリープを解除する」に**チェック**
7. **設定**タブ
   - 「スケジュールされた時刻にタスクを開始できなかった場合、すぐに開始する」にチェック
   - 「既にタスクが実行中の場合: 新しいインスタンスを開始しない」
8. OK

## 動き始めたあと

### 毎日やること(3分)

**スマホで:**
1. GitHubアプリで `PROGRESS.md` → 昨日何が進んだか
2. `IMPROVEMENTS_LOG.md` → Claudeが保留した項目
   - 「要外部AI」→ 書いてあるプロンプトをGeminiに投げて画像を作る
   - 「要ユーザー判断」→ 決めて `REQUESTS.md` に書く
3. `REQUESTS.md` → Claudeからの返事や質問に答える

**PCで:**
```
natsu play      遊んでみる
natsu req "..."  気になったところを投げる
```

### たまにやること

```
natsu doctor    週1くらい。ちゃんと動いているか確認
natsu status    今の状態と次回実行時刻
```

### 止めたくなったら

```
natsu pause     止める(その時点のゲームが開きます)
natsu resume    再開する
```

### 何かおかしくなったら

```
natsu undo      直前の改変を取り消す
```

もっと前に戻したいときは `git log --oneline` で履歴を見て、
戻したい地点のIDを控えて `git revert <ID>`。

さらに、`snapshots` フォルダに毎日1つ、その日の最初の状態が
自動で保存されています。最悪これを手でコピーすれば戻せます。

---

## 仕込んである保険

自分で気づかなくても、以下は自動で処理されます。

- **ゲームファイルが壊れたら自動で直前の状態に戻す**(実行のたびに確認)
- **毎日1回バックアップ**(`snapshots` フォルダ、30日で自動削除)
- **異常終了して固まっても2時間後に自動解除**
- **利用制限中は何もせず終了**し、明けた最初のタイミングで自動再開
- **スリープ中でもPCを起こして実行**
- **ログが増えすぎたら古い分を自動削除**

## 困ったら

`natsu doctor` の画面をスクショしてClaudeのチャットに送ってください。
`auto_run.log` の中身を貼ってもらえれば、だいたい原因が分かります。
