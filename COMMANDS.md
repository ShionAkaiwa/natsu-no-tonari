# コマンド一覧

PowerShellで `D:\natsu-project` に移動して `.\natsu.bat <コマンド>`。
ステップ9の設定を済ませていれば、どこからでも `natsu <コマンド>` でOK。

---

## 遊ぶ

| コマンド | 何が起きるか |
|---|---|
| `natsu play` | **止めずに**今のゲームを開いて遊ぶ。裏では改善が進み続ける |
| `natsu pause` | 自動改善を止めて、その時点のゲームを開く。作業中なら終わるまで待ってから開く |

`play` はコピーを開くので、Claudeが編集中の壊れかけファイルを掴むことはありません。
毎回同じ `play_now.html` を開くので、ゲーム内のセーブデータも引き継がれます。

## 止める・再開する

| コマンド | 何が起きるか |
|---|---|
| `natsu pause` | 一時停止。以降タスクが起動しても何もせず終了する |
| `natsu resume` | 再開。**何日止まっていても、制限が明けていればここから再開できる** |
| `natsu now` | 次の定期実行を待たず、今すぐ1回だけ走らせる |

制限に当たっている間は自動的に「何もせず終了」を繰り返すだけなので、
放っておけば制限が明けた最初のタイミングで勝手に再開します。
`resume` が要るのは、自分で `pause` したときだけです。

## Claudeに要望を伝える

| 方法 | やり方 |
|---|---|
| PCで一言 | `natsu req "花火のシーンをもっと長くしてほしい"` |
| PCで長文 | `natsu talk` → メモ帳が開くので書いて保存 |
| **スマホから** | GitHubアプリで `REQUESTS.md` を開いて鉛筆アイコン → 編集 → Commit |

書いた要望は次の実行で最優先で処理され、`REQUESTS.md` の
その要望の下にClaudeが返事を書きます。返事はGitHubからも読めます。

## 状況を見る

| コマンド | 何が見えるか |
|---|---|
| `natsu status` | 稼働中か停止中か、未読の要望が何件あるか、直近のログ |
| `natsu log` | 実装された改善と、判断が保留された項目 |
| `git log --oneline` | 変更の履歴一覧 |
| `git diff HEAD~1` | 直前の変更の中身 |

`PROGRESS.md`(やったこと)と `IMPROVEMENTS_LOG.md`(保留したこと)は
GitHubからスマホでも読めます。

## 取り消す

| コマンド | 何が起きるか |
|---|---|
| `natsu undo` | 直前の自動改善を取り消す(確認あり) |
| `git revert <ID>` | もっと前の変更を狙って取り消す。IDは `git log --oneline` で確認 |


## 困ったとき

| コマンド | 何が起きるか |
|---|---|
| `natsu doctor` | 環境をまるごと診断。全部緑なら正常 |
| `natsu snapshot` | 今のゲームを手動でバックアップ |
| `natsu undo` | 直前の自動改善を取り消す(確認あり) |

`natsu doctor` は「Node.js / Git / Claude Code が入っているか」
「GitHubに繋がるか」「ゲームファイルが壊れていないか」
「定期実行が登録されているか」を一度に確認します。
週1くらいで打っておくと安心です。

---

## 自動で守られていること

自分で気にしなくても、以下は勝手に処理されます。

- ゲームファイルが壊れたら自動で直前の状態に戻す
- 毎日1回 `snapshots` フォルダにバックアップ(30日で自動削除)
- 異常終了して固まっても2時間後に自動解除
- 利用制限中は何もせず終了し、明けたら自動再開
- スリープ中でもPCを起こして実行

## 毎日の流れ(参考)

1. スマホでGitHubを開く → `PROGRESS.md` で昨夜の成果を見る
2. `IMPROVEMENTS_LOG.md` を見る
   - 「要外部AI」→ 書いてあるプロンプトをGeminiに投げて画像を作る
   - 「要ユーザー判断」→ 決めた内容を `REQUESTS.md` に書く
3. `REQUESTS.md` でClaudeからの返事や質問に答える
4. PCで `natsu play` して実際に遊ぶ
5. 気になったところを `natsu req` で投げる
